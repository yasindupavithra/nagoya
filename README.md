# Nagoya Auto Auction – Next.js + Firebase Serverless Platform

A buyer-centric automotive marketplace for Sri Lanka's largest auto auction, built entirely serverless with Next.js 14, React, Firestore, Firebase Storage, and Cloud Functions.

## 🎯 Features

### Buyer Pages
- **Home** (`/`) – Hero with advanced quick search filters and featured inventory
- **Inventory** (`/inventory`) – Full vehicle catalog with dynamic filters (price, year, fuel type, mileage)
- **Vehicle Detail** (`/vehicle/[id]`) – Image gallery, specs, verified inspection report, lease calculator, inquiry form

### Admin Pages
- **Login** (`/admin/login`) – Email/password authentication
- **Dashboard** (`/admin`) – Active listings, SOLD count, new inquiries
- **Inventory Management** (`/admin/inventory`) – Search, toggle SOLD status, delete listings
- **Add Vehicle** (`/admin/inventory/add`) – Multi-image upload → Firebase Storage → Firestore save

### Backend
- **Cloud Functions**
  - `autoFillVehicleSpecs` – Auto-populate vehicle specs on document creation
  - `onInquiryReceive` – Send email notification to owner when inquiry arrives
- **Firestore Collections**
  - `vehicles` – Full catalog with images, specs, prices
  - `inquiries` – Buyer contact requests
- **Firebase Rules** – Public read for vehicles/images, authenticated admin writes, public inquiry creation

## 🚀 Setup & Local Development

### 1. Install Dependencies
```bash
npm install
cd functions && npm install && cd ..
```

### 2. Configure Firebase
Create or update `firebaseConfig.js`:
```javascript
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
  appId: "YOUR_APP_ID"
};
export default firebaseConfig;
```

Get these from Firebase Console → Project Settings → Web.

### 3. Update `.firebaserc`
Replace `YOUR_PROJECT_ID`:
```json
{
  "projects": {
    "default": "YOUR_PROJECT_ID"
  }
}
```

### 4. Deploy Firebase Rules
```bash
firebase deploy --only firestore:rules,storage:rules
```

### 5. Deploy Cloud Functions
Update email credentials in `functions/src/index.ts` for notifications, then:
```bash
firebase deploy --only functions
```

### 6. Run Development Server
```bash
npm run dev
```
Visit `http://localhost:3000`

## 🔐 Firebase Security Rules

### Firestore Rules (`firestore.rules`)
```
- `/vehicles/{vehicle}` → Read: Public, Write: Authenticated
- `/inquiries/{inquiry}` → Read: Authenticated, Create: Public
```

### Storage Rules (`storage.rules`)
```
- `/vehicles/{imageId}` → Read: Public, Write: Authenticated
```

### 2. Deploy

## 📋 Admin Setup

### Create Admin User
1. Go to [Firebase Console](https://console.firebase.google.com)
2. Authentication → Users → Add User
3. Create email/password for owner
4. Use those credentials to log in at `/admin/login`

### Add First Vehicle
1. Sign in at `/admin/login`
2. Go to Admin Dashboard → Add Vehicle
3. Fill form + upload images
4. Images upload to Storage, record saves to Firestore with image URLs

## 💾 Firestore Collection Creation

Firestore auto-creates collections on first document write, so:
1. Add first vehicle → `vehicles` collection created
2. Submit first inquiry → `inquiries` collection created

Or manually create in Console if preferred.
